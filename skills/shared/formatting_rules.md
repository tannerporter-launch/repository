# Formatting rules (Tanner's, follow exactly)

These rules apply to every output the pipeline produces: Notion content, code,
code comments, commit messages, and chat. Every skill in this bundle must
follow them.

## Rule: Em dashes are banned

- Never use an em dash anywhere, in any output, code comment, or Notion content.
- Use a period, a comma, the word "to", or parentheses instead.

## Rule: Bullets stay flat and paste-safe

- No sub-bullets. Every list is a single flat level.
- Each bullet starts with a dash on its own line.
- Bullets must survive a plain-text paste, so do not rely on nesting or indentation to carry meaning.

## Rule: Section titles lead with the tool or feature name

- Training and section titles start with the specific tool or feature name so they are scannable in a list.
- Example: "yt-dlp extraction" not "How extraction works".
- Example: "Notion table of contents" not "Adding navigation".

## Rule: Earn the space

- No filler. Every section and every bullet must add something a reader would act on or reference.
